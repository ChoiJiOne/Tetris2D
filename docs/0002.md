## Visual Studio 솔루션 생성 자동화를 하는 이유는 무엇인가요?

Tetris2D 프로젝트의 Visual Studio 솔루션 생성을 자동화 하는 이유는 크게 다음 두 가지 때문입니다.  

1. 다양한 환경에서의 구현 작업 진행  
2. 빌드 자동화 작업  

첫 번째의 경우, 집에서 작업할 때가 대부분이지만 특별한 경우에 밖에서 노트북으로 작업하는 경우가 있는데, 처음에 생성한 솔루션을 그대로 사용할 경우 Windows SDK의 버전이 다르면 빌드가 되지 않는 경우가 있습니다. 따라서, 이러한 상황을 미리 막기 위해 Visual Studio 솔루션 생성 자동화를 구현합니다.  
    
두 번째의 경우, 조엘 테스트 항목에도 있듯이 이는 일일 빌드를 자동화 하기 위한 것으로 자동으로 Visual Studio 솔루션을 생성한 뒤에 그 솔루션과 `MSBuild.exe`를 사용하여 일일 빌드 자동화를 구현하기 위함입니다.  
<br><br>


## Visual Studio 솔루션 생성 자동화 구현은 무엇을 이용해서 하나요?

Visual Studio 솔루션 생성 자동화의 경우 많은 오픈 소스 프로젝트가 `CMake`를 사용합니다.  
하지만, 이 프로젝트의 경우 단 하나이지만 무조건적이고 강력한 조건을 만족해야 합니다.  
그것은 **"쉽고 간단하게 사용할 수 있어야 한다."** 입니다.  
이 조건을 만족하는 것으로는 [Premake](https://premake.github.io/)가 있습니다.  
따라서, 이 Tetris2D 프로젝트에서 Visual Studio 솔루션 생성 자동화를 수행하기 위해서는 Premake5를 사용합니다.    
<br><br>


## Visual Studio 솔루션 생성을 생성하는 방법

프로젝트 내에 `GenerateProjectFiles.bat` 파일을 실행할 경우, `Tetris2D` 폴더 하위에 `Tetris2D.sln` 파일을 얻을 수 있습니다.
